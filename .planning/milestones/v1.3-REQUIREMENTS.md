# Requirements: My SuperWhisper

**Defined:** 2026-03-24
**Core Value:** Frictionless voice-to-text that produces clean, well-formatted Spanish text — press a key, speak, press again, and polished text appears where you're typing. Local STT + Haiku cleanup.

## v1.3 Requirements

Requirements for Settings UX milestone. Each maps to roadmap phases.

### Window Behavior

- [x] **WIN-01**: La ventana de Settings permanece abierta cuando el usuario hace click fuera de ella
- [x] **WIN-02**: La ventana de Settings se cierra solo con el botón X o Cmd+W
- [x] **WIN-03**: Al cerrar Settings, el dock icon desaparece y el foco vuelve a la app anterior
- [x] **WIN-04**: La ventana de Settings acepta input de teclado (TextFields, hotkey recorder)

### Settings UI

- [x] **UI-01**: Settings usa SwiftUI Form con `.formStyle(.grouped)` como layout principal
- [x] **UI-02**: Settings agrupados en secciones lógicas: Grabación, Transcripción, Vocabulario, Sistema
- [x] **UI-03**: El hotkey recorder usa `KeyboardShortcuts.Recorder` nativo de SwiftUI
- [x] **UI-04**: El selector de micrófono funciona como `Picker` SwiftUI con los dispositivos disponibles
- [x] **UI-05**: Las correcciones de vocabulario son editables en una `List` con `TextField` inline
- [x] **UI-06**: Los toggles (launch at login, pause playback, maximize volume) funcionan y persisten estado
- [x] **UI-07**: El botón de API key abre el panel existente de ingreso de clave

## Future Requirements

### Deferred from v1.3

- **WIN-05**: Posición de la ventana persistida entre sesiones (frameAutosaveName)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Push-to-talk mode | Fuera de scope v1.3, considerar para v1.4+ |
| Configurable LLM cleanup aggressiveness | Fuera de scope v1.3 |
| Tabs en Settings | Solo 7 settings — single-pane con secciones es suficiente |
| SwiftUI Settings scene / openSettings | Roto en macOS 26 Tahoe, requiere workarounds para menubar apps |
