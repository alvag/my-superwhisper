  <how-to-verify>
    Execute each scenario below in order. Report results as PASS/FAIL/PARTIAL with notes.

    **IMPORTANT TESTING RULES (from research pitfalls):**
    - Let media play for 5+ seconds before triggering recording hotkey
    - For "nothing playing" test: quit Music.app completely (verify with `pgrep -x Music` returning nothing)
    - For toggle OFF test: first verify toggle ON works, then switch OFF, then test

    ---

    ## SECTION A: Spotify (primary player)

    **A1. Pause on record start**
    1. Open Spotify, play any song, let it play 5+ seconds
    2. Press recording hotkey (Option+Space)
    3. Expected: Spotify pauses within ~200ms, overlay appears
    Result: PASS / FAIL

    **A2. Resume on record stop**
    1. While recording (from A1), press hotkey again to stop
    2. Expected: Spotify resumes playing, overlay shows processing spinner
    Result: PASS / FAIL

    **A3. Resume on Escape cancel**
    1. Play song in Spotify, press recording hotkey
    2. Press Escape to cancel
    3. Expected: Spotify resumes, beep sounds, overlay hides
    Result: PASS / FAIL

    **A4. Already-paused Spotify (no double-toggle)**
    1. Play song in Spotify, then PAUSE it manually
    2. Press recording hotkey, speak, press hotkey again
    3. Expected: Spotify stays paused after recording completes (was NOT playing before)
    Result: PASS / FAIL

    ---

    ## SECTION B: Apple Music

    **B1. Pause on record start**
    1. Open Apple Music, play any song, let it play 5+ seconds
    2. Press recording hotkey
    3. Expected: Apple Music pauses
    Result: PASS / FAIL

    **B2. Resume on record stop**
    1. While recording (from B1), press hotkey again
    2. Expected: Apple Music resumes
    Result: PASS / FAIL

    ---

    ## SECTION C: YouTube in Safari

    **C1. Pause on record start**
    1. Open Safari, go to youtube.com, play any video, let it play 5+ seconds
    2. Press recording hotkey
    3. Expected: YouTube video pauses
    Result: PASS / FAIL

    **C2. Resume on record stop**
    1. While recording (from C1), press hotkey again
    2. Expected: YouTube video resumes
    Result: PASS / FAIL

    **C3. Background tab behavior (informational)**
    1. Play YouTube video, switch to a different Safari tab
    2. Press recording hotkey, then stop
    3. Note: Does YouTube resume? (PARTIAL result acceptable)
    Result: PASS / PARTIAL / FAIL + notes

    ---

    ## SECTION D: Edge Cases

    **D1. Nothing playing — Music.app guard (CRITICAL)**
    1. Quit ALL media apps: Spotify, Music, Safari, VLC, Chrome, Firefox
    2. Verify: run `pgrep -x Music` in Terminal — should return nothing
    3. Verify: run `pgrep -x Spotify` — should return nothing
    4. Press recording hotkey, speak briefly, press hotkey again
    5. Expected: Recording completes normally, Music.app does NOT launch
    Result: PASS / FAIL (FAIL = release blocker)

    **D2. Rapid double-tap**
    1. Open Spotify, play a song
    2. Press recording hotkey twice as fast as possible (within ~100ms)
    3. Expected: After everything settles, Spotify is in the same state as before (playing)
    4. Note any error notifications or overlay glitches
    Result: PASS / FAIL + notes

    **D3. Settings toggle OFF — no media events**
    1. Open Spotify, play a song
    2. Open MyWhisper Settings, UNCHECK "Pausar reproduccion al grabar"
    3. Close Settings
    4. Press recording hotkey, speak, press hotkey again
    5. Expected: Spotify continues playing throughout recording (never pauses)
    Result: PASS / FAIL

    **D4. Settings toggle persists after restart**
    1. With toggle OFF (from D3), quit MyWhisper completely
    2. Relaunch MyWhisper
    3. Open Settings, verify checkbox is still unchecked
    4. Result: PASS / FAIL

    **D5. Re-enable toggle**
    1. In Settings, CHECK "Pausar reproduccion al grabar" again
    2. Play song in Spotify
    3. Press recording hotkey
    4. Expected: Spotify pauses (feature is re-enabled)
    Result: PASS / FAIL

    ---

    Report ALL results. For any FAIL, describe exactly what happened.
