## Phase Details

### Phase 9: Window Lifecycle Foundation
**Goal**: La ventana de Settings permanece abierta cuando el usuario interactúa fuera de ella y gestiona correctamente el ciclo de vida de activación
**Depends on**: Phase 8 (v1.2 complete)
**Requirements**: WIN-01, WIN-02, WIN-03, WIN-04
**Success Criteria** (what must be TRUE):
  1. Usuario abre Settings desde el menubar, hace click en el escritorio o en otra app, y la ventana de Settings permanece visible
  2. Usuario puede cerrar Settings con el botón X o Cmd+W, y el dock icon desaparece inmediatamente
  3. Al cerrar Settings, el foco vuelve automáticamente a la app que estaba activa antes de abrir Settings
  4. Usuario puede tipear en un TextField dentro de Settings (input de teclado funciona correctamente)
**Plans:** 2 plans
Plans:
- [x] 09-01-PLAN.md — Crear SettingsViewModel y SettingsView placeholder con hotkey recorder + toggle
- [ ] 09-02-PLAN.md — Reescribir SettingsWindowController: NSPanel a NSWindow + NSHostingController + activation policy
**UI hint**: yes

### Phase 10: Settings SwiftUI UI
**Goal**: Todos los settings del app son accesibles y funcionales desde un SwiftUI Form con layout agrupado estilo System Preferences
**Depends on**: Phase 9
**Requirements**: UI-01, UI-02, UI-03, UI-04, UI-05, UI-06, UI-07
**Success Criteria** (what must be TRUE):
  1. Settings muestra cuatro secciones agrupadas (Grabación, API, Vocabulario, Sistema) con layout inset-grouped idéntico al de System Settings de macOS
  2. Usuario puede cambiar el hotkey global usando el KeyboardShortcuts recorder nativo y el cambio persiste tras reiniciar la app
  3. Usuario puede seleccionar su micrófono desde un Picker, activar/desactivar los tres toggles (launch at login, pause playback, maximize volume) y todos los cambios persisten en UserDefaults
  4. Usuario puede agregar, editar y eliminar correcciones de vocabulario desde la lista inline sin salir de Settings
  5. Usuario puede abrir el panel de API key desde el botón en Settings y la clave se guarda correctamente en Keychain
**Plans:** 1/1 plans complete
Plans:
- [ ] 10-01-PLAN.md — Expandir SettingsView con 4 secciones completas (Grabacion, API, Vocabulario, Sistema) + VocabularyEntry Identifiable
**UI hint**: yes
